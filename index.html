Course Agenda

  • Project skeleton

    Work always with a base skeleton. This skeleton will never contain business logic, but all what you'll need to deploy the project anywhere

    • Layers definition
    • Install Symfony4. The framework will work, only, as an entry point
    • Install dependencies. Composer to the rescue
    • Command Bus as the core member of the project
    • Commands, command handlers and middleware classes

  • Docker basics

    We will learn all the needed docker skills you need to deploy your application.

    • Docker basic features
    • Docker composer
    • Creating a base Dockerfile

  • ReactPHP Server

    Our project will run with ReactPHP as a Server. That will bring us deploy simplicity and an insane performance improvement.

    • Check the PHP-PM project
    • Check the ReactPHP Symfony server
    • Discussion and server installation

  • Define your domain

    Always with a simple but complete project in front of us, we will start looking at our model. In this part we will not load any external dependency like Redis or Mysql. Only our own classes. After this module, our project will be working properly without any persistence layer and completely synchronous.

    • Needs analysis
    • Model classes implementation
    • I/O repositories interfaces and in-memory implementation
    • Full testing of our needs, both unit and functional

  • Dig into I/O dependencies

    Once our project is properly working, then let's add our persistence layers, like Redis or Mysql. We will see that adding tests coverage to these classes is extremely trivial and fast. We will see how easy is to change from one adapter to another.

    • Redis adapter for our cache layer
    • Mysql adapter for our persistence layer
    • I/O repositories interfaces and in-memory implementation
    • Full testing of our needs, both unit and functional

  • Dig into asynchronous feature

    Using asynchronous console consumers will help you to split, literally, between write and read operations. Even across different infrastructure members. We will turn our application asynchronous, adding test coverage and some tips and tricks.

    • Adding asynchronous middleware for our write operations
    • RabbitMQ adapter
    • Adding testing coverage for our asynchronous writes
